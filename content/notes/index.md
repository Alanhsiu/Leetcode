My running stream of captured notes. Each one is timestamped (explicit in
frontmatter, or derived from its first git commit) and sorted **newest first**.
Use the **Newest / Oldest** toggle below, or see everything chronologically on the
[Timeline](/timeline).

## How a note gets here

Two commands, from anywhere with the repo checked out:

```bash
npm run note -- "What is a service mesh"   # scaffold + open in $EDITOR
# ...write...
npm run publish                            # commit + push
```

On a phone (no CLI), add a markdown file straight from the GitHub web editor — the
build derives its timestamp from the commit date, so it still sorts correctly. See
the README section **“How I capture a note”** for the exact template.

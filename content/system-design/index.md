The system-design interview is open-ended on purpose. You're not graded on a single
"right" answer — you're graded on how you **scope an ambiguous problem, make trade-offs
out loud, and reason about scale**. These notes give you a repeatable structure plus the
core building blocks to reach for.

Start with **[A framework for the interview](/system-design/framework)** for the
minute-by-minute plan, then skim **[Building blocks](/system-design/building-blocks)** so
the vocabulary is at your fingertips.
